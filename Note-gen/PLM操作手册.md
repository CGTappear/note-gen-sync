### 9.4 零件号生命周期

| 零件号类型 | 生命周期状态 |
|---|---|
| 项目零件号 | **提交** → **归档** → **冻结** → **报废** |
| 产品零件号（10） | **提交** → **归档** → **冻结** → **报废** |
| 软件半成品零件号（20.01） | **提交** → **归档** → **冻结** → **报废** |
| 硬件半成品零件号（20.02） | **提交** → **归档** → **冻结** → **报废** |
| 机械半成品零件号（20.03） | **提交** → **归档** → **冻结** → **报废** |
| 包装半成品零件号（20.04） | **提交** → **归档** → **冻结** → **报废** |
| 硬件子零件号（30.02） | **提交** → **归档** → **认可** → **冻结** → **报废** |
| 机械子零件号（30.03） | **提交** → **归档** → **冻结** → **报废** |
| 胶水子零件号（30.04） | **提交** → **归档** → **冻结** → **报废** |
| 包装子零件号（30.05） | **提交** → **归档** → **冻结** → **报废** |

**状态说明：**
- **提交：** 零件号新创建且未审批时处于提交状态
- **归档：** 零件号创建好并且审批完成后处于归档状态
- **认可：** 零件号处于归档状态且测试验证通过后处于认可状态（仅硬件子零件号）
- **冻结：** 已归档的零件号因特殊原因暂时停止使用时处于冻结状态
- **报废：** 已归档的零件号后续不再使用时处于报废状态

### 9.5 零件号创建流程

**创建流程：** 选择分类 → 填写零件信息 → 保存

1. 选择"物料管理"模块下的"物料库"
2. 在物料类别列表中选择需要创建的零件类别，然后点击"新增"
3. 在弹出的界面中按要求填写物料属性，填写完成后点击"保存"即可关闭该界面。（注意：创建硬件物料时需要启用保质期管理）
4. 在物料库所属分类界面点击"刷新"即可看到刚刚创建的零件号
5. 如果该物料后续要升版本，打开物料属性界面，点击"升大版"即可使该物料从"归档"状态转换为"提交"状态，提交状态下可编辑物料属性，修改版本号后点击"保存"，最后重新发起物料审批流进行审批即可完成升版操作。

### 9.6 零件号审批

**审批流程：** 启动流程 → 领导审批 → 零件号归档

1. 零件号创建好之后勾选需要审批的零件号，然后点击"启动流程"
2. 在弹出的对话框中勾选要审批的零件号，然后点击"提交"，即可发起审批流。零件审批流为一级审批，相应职能主管审批后即转换到"归档"状态
3. 各职能主管在"待办流程"中即可看到团队成员提交的零件号审批流
4. 双击可以打开该审批流，在弹出的审批窗口中可以双击查看待审批对象的详情，如果确认无误，选择"审核通过"，然后点击"提交"该审批流即可审批完成。如果审批对象存在问题，也可以选择"驳回重审"或者"终止流程"
5. 审批完成后零件号生命周期处于"归档状态"，物料列表中图标颜色显示为绿色

### 9.7 零件号属性修改

**修改流程：** 填写变更单 → 启动流程 → 领导审批 → 变更实施 → 变更完成

1. 零件号的属性修改是以变更的形式实现的，单击"变更单列表"
2. 在弹出的变更列表窗口选中"文件夹"视图下的"物料变更"文件夹，然后点击"新增"
3. 按要求填写变更单属性，然后单击"新增设计变更对象"，在弹出的窗口中展开"Material（物料）"然后选中想要变更的零件号，单击"返回数据"该零件号即展示在变更对象列表中，然后点击左上角"保存"即完成了变更单的创建
4. 返回变更单列表界面，点击"刷新"即可显示刚刚创建的变更单
5. 选中需要审批的变更单点击"提交"便启动了变更单审批流
6. 此时在变更审批人的待办流程中界面即可看到变更任务
7. 如果审批人对变更没有异议的话单击"审核通过"，该流程将流转到变更负责人环节实施变更
8. 变更负责人待办流程界面会收到变更流程
9. 打开变更流程后在弹出的界面中双击打开需要变更的对象
10. 跳转到需要修改的零件号界面后修改相关属性，然后点击"保存"并关闭零件属性界面
11. 返回变更单界面后，点击"标记完成"，然后选择"确认继续"，最后点击"提交"完成变更的实施
12. 此时在变更单列表界面看到该变更单已完成。在该零件号的"相关变更"界面也可以看到相关变更记录。至此，该零件号属性的修改工作全部完成

### 10.4 BOM创建

1. 打开BOM管理模块下的"设计BOM库"
2. 选择文件夹视图下的"产品总成"文件夹，然后点击"新增"
3. 在BOM创建界面补充BOM属性，填写完成后点击"保存"，然后关闭该界面
4. 在设计BOM库产品总成分类下面选择需要审批的产品BOM，然后点击"启动流程"，在弹出的对话框中双击相应的BOM审批流程，在弹出的界面点击"提交"即可启动BOM审批流程
5. BOM当前采用多实例并存的方式去管理。后续如果有BOM升版需求，需要单独创建BOM，多个BOM并行存在，通过产品BOM来体现子BOM间的关联关系

### 10.5 BOM审批

1. 职能领导在待办流程下可以看到提交的BOM审批流程
2. 双击审批流之后在弹出的对话框中可以查看审批对象是否正确，如无异议，选择"审核通过"后点击"提交"即可完成审批。当前产品BOM采用**二级审批**，半总成BOM采用**一级审批**，各职能主管审批后BOM状态变为"归档"状态

### 10.6 BOM修改

1. BOM的属性修改是以变更的形式实现的，单击"变更单列表"
2. 在变更列表"文件夹"视图下，选中"BOM变更"文件夹，然后点击"新增"
3. 在变更单新增界面填写变更单属性，然后点击下方"新增设备变更对象"，按步骤选中需要变更的BOM，最后点击"保存"即可完成BOM变更单的创建
4. 返回变更单列表界面，在BOM变更单文件夹下选中新创建的变更单点击"提交"即可发起审批流
5. 此时在变更审批人的待办流程中界面即可看到变更任务
6. 如果审批人对变更没有异议的话选择"审核通过"，然后点击"提交"该流程将流转到变更负责人环节实施变更
7. 变更负责人待办流程界面会收到变更流程
8. 打开变更流程后在弹出的界面中双击打开需要变更的对象
9. 跳转到需要修改的BOM界面后先点击"检出"，然后修改BOM，然后再点击"保存"，最后点击"检入"并关闭BOM界面
10. 返回变更单界面后，点击"标记完成"，然后选择"确认继续"，最后点击"提交"完成变更的实施
11. 此时在变更单列表界面看到该变更单已完成。在该BOM的"相关变更"界面也可以看到相关变更记录。至此，该BOM的修改工作全部完成

### 10.7 BOM发布

#### 10.7.1 硬件发布

硬件主管在收到硬件半成品零件号的发布审批流时，需确认下列输出物是否已上传PLM，以及相关版本信息是否正确：

- [ ] a. 对应版本的硬件BOM和STUFF文件

- [ ] c. 对应版本的PCB单板文件
- [ ] d. 对应版本的PCB拼板文件

### 10.8 BOM下推

1. 打开已经归档的总成(10)BOM，点击"启动流程"，在弹出的对话框中选择"BOM下推审批流程"
2. 双击该审批流可跳转到流程发起界面，勾选需要审批的对象，然后点击"提交"即可发起审批流
3. 系统主管在"工作中心"→"待办流程"中可以看到需要审批的流程
4. 双击可以打开该审批流，在弹出的审批窗口中可以双击查看待审批对象的详情，如果确认无误，选择"审核通过"，然后点击"提交"该审批流即可审批完成。如果审批对象存在问题，也可以选择"驳回重审"或者"终止流程"

## 13. 工时填报

### 13.1 概述

工时填报模块主要用于统计研发人员在各项目中的工时消耗情况，用于项目中期或项目结束后的复盘统计工作。

**填报要求：**
1. 工时填报要求的最小颗粒度为"1天/条"
2. 不可以将同一项目多天的任务填报在一条记录中
3. 月底前需要将本月的工时都填报完成

### 13.2 模块位置

### 13.3 本人工时填报流程

1. 打开"工时管理（List）"界面后，选择自己所在的分组，然后鼠标左键选中自己的名字，点击"新增"按钮
2. 在弹出窗口界面中填写相关信息后点击"保存"按钮即可创建一条填报记录，然后关闭该窗口。（注意：工作时长按照钉钉中的打卡记录时间填写，出差的工作时长需要自行统计填写）
3. 由于PLM非实时性系统，在"工时管理（List）"界面点击"刷新"后才能看到刚刚填报的记录
4. 为了防止填报之后记录被误操作，选中需要保护的记录，然后点击"状态转换"到"归档"。那么该记录就处于不可编辑状态
